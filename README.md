# Discord生成AIボット 最終課題企画

## 採用する企画

候補3の「チャンネル議事録Bot」を実装する。

BotがDiscordチャンネルの実際の会話履歴を読み取り、Geminiを使って次の項目へ整理する。

- 話題
- 決定事項
- 担当作業
- 期限
- 保留事項

生成AIへ送信するときはDiscordの表示名を使わず、発言者を `参加者1`、`参加者2` のように置き換える。

### 現在実装しているコマンド

- `/議事録`：現在のチャンネルの直近30件から議事録を作る
- `/議事録 件数:50`：5〜100件の範囲で読み取る件数を指定する
- `/help`：使い方を表示する

### Discordでなければできない点

Botへの質問文だけをAIへ送るのではなく、複数メンバーが普段使っているDiscordチャンネルの会話履歴を取得する。会議専用の別サービスへ全員が文章をコピーすることなく、その場で決定事項や作業を共有できる。スレッド内で実行した場合は、そのスレッドの会話だけを議事録にできる。

## 起動方法

1. このフォルダで `npm install` を実行する
2. `.env.example` を参考に `.env` を作成する
3. Discord Developer PortalでBotの `Message Content Intent` を有効にする
4. Botへ次の権限を付けてサーバーへ招待する
   - View Channels（チャンネルを見る）
   - Send Messages（メッセージを送信）
   - Read Message History（メッセージ履歴を読む）
   - Use Application Commands（アプリコマンドを使用）
5. `npm start` を実行する

### Dockerで起動する場合

`.env` を作成してから、次を実行する。

```bash
docker compose -f docker.compose.yml up --build
```

停止する場合：

```bash
docker compose -f docker.compose.yml down
```

起動後、ブラウザで `http://localhost:3000` を開くと稼働確認ができる。

環境変数：

```text
DISCORD_TOKEN=Discord Botのトークン
GEMINI_API_KEY=Gemini APIのキー
GEMINI_MODEL=gemini-3-flash-preview
```

## Render設定

- Runtime：Node
- Build Command：`npm install`
- Start Command：`npm start`
- 環境変数：`DISCORD_TOKEN`、`GEMINI_API_KEY`、必要なら `GEMINI_MODEL`

RenderのURLへアクセスすると `Discord Meeting Minutes Bot is running` と表示される。

---

以下は企画検討時の記録。

## 課題の必須条件

- Discord上で動くチャットボットにする
- 生成AI（Gemini）を利用する
- 実生活や共同作業の役に立つ機能を持たせる
- `/help` で使い方を表示する
- Render上で常時起動できる構成にする
- 個人情報は扱わない

## 授業で作成したGeminiBotとの差別化

授業の `GeminiBot` は、スラッシュコマンドを受け取ると固定プロンプトをGeminiへ送り、生成結果を1件返す構成になっている。

最終課題では、プロンプトだけを変更するのではなく、次のようなDiscord固有の情報や操作を組み合わせる。

- チャンネル内の複数メッセージを収集して整理する
- 複数メンバーの意見を比較する
- ボタンや投票で次の行動につなげる
- Discordのチャンネルやスレッド単位で結果を返す

## 「Discordのチャットボットでなければできないこと」の定義

単にDiscordから生成AIへ質問できるだけでは、この条件を満たしたとは言いにくい。Webサイトの入力欄でも同じ機能を作れるためである。

この作品では、Discordにすでに存在する次の要素を機能の中心にする。

- コマンドを実行したチャンネルから議論用スレッドを作る
- スレッド内で複数メンバーが投稿したメッセージを取得する
- 誰がどの意見を投稿したかではなく、議論全体の論点をAIで整理する
- 整理結果をスレッドへ返し、Discordのボタンで参加者が投票する
- 投票結果と未決定事項を同じスレッドへ掲載し、参加メンバーへ通知する

つまり、`Discord上の会話 → AIによる整理 → Discord上の合意形成` を一つの流れとして実現する。すでに参加者が集まり、会話しているDiscordサーバーだからこそ役立つ機能にする。

## 候補

### 1. Discord議論ファシリテーターBot（第一候補）

コマンドを実行したチャンネルに議論専用スレッドを作成する。参加者は普段どおりスレッドへ意見を書き込み、Botはそのメッセージ履歴を取得する。AIが論点・提案・利点・心配点・未決定事項を整理し、最後にDiscordのボタン投票を作って、話し合いを実際の決定につなげる。

想定コマンド：

- `/議題 タイトル:...`：現在のチャンネルに議論用スレッドを作る
- `/整理`：現在のスレッドにある複数人の投稿を取得し、AIが分類・要約する
- `/投票`：整理された案を選択肢にして、Discordのボタン投票を投稿する
- `/結果`：得票数、決定した案、未決定事項をスレッドへ表示する
- `/help`：使い方を表示する

役立つ場面：文化祭の企画、グループ課題の方針、旅行先、サークル活動の決定。

Discordでなければできない点：チャンネルからのスレッド作成、スレッド参加者の自然な会話、メッセージ履歴の取得、Discordユーザーによるボタン投票、参加者への通知を連動させる。AIチャットの画面で一人が質問するだけでは実現できない、複数人の合意形成を支援する。

実装難易度：中

### 2. グループ課題タスク分解Bot

チャンネルに書かれた課題の説明と締切から、AIが必要な作業を分解する。メンバーはボタンで担当を選び、Botが未担当・進行中・完了を一覧表示する。

想定コマンド：

- `/課題登録 説明:... 締切:...`
- `/タスク作成`：AIが作業候補を分解する
- `/担当一覧`
- `/完了 タスク:...`
- `/help`

役立つ場面：授業のグループ制作、イベント準備、サークル活動。

工夫：生成AIによるタスク分解と、Discordメンバーによる担当・進捗管理を組み合わせる。生成だけで終わらず、チームで継続利用できる。

実装難易度：中〜高

### 3. チャンネル議事録Bot

指定された範囲のチャンネル投稿を読み、決まったこと・担当・期限・保留事項に分けて議事録を作る。要約結果から担当者別の確認メッセージも作成する。

想定コマンド：

- `/議事録 件数:50`
- `/決定事項`
- `/未解決`
- `/help`

役立つ場面：オンライン会議後の確認、欠席者への共有、長い相談の振り返り。

工夫：単なる要約ではなく、Discordの会話を「決定」「担当」「期限」「保留」に分けて、次にすることを明確にする。

実装難易度：中

### 4. 質問重複防止・FAQ作成Bot

質問チャンネルの過去メッセージを調べ、似た質問と回答候補をAIが提示する。解決済みになった会話からFAQを作り、後から検索できるようにする。

想定コマンド：

- `/質問 内容:...`
- `/解決`：現在のやり取りをFAQ候補にする
- `/faq検索 キーワード:...`
- `/help`

役立つ場面：授業、部活動、新入生案内、開発チームの質問チャンネル。

工夫：AIの回答だけに頼らず、そのDiscordサーバー内で実際に解決した情報を再利用する。十分に作る場合はPostgreSQLも組み合わせられる。

実装難易度：高

### 5. 学習会クイズ・弱点分析Bot

学習したい資料やテーマからAIが問題を作り、Discord上で複数人が回答する。正答率を集計し、グループが苦手な分野を分析して次の復習問題を出す。

想定コマンド：

- `/クイズ作成 テーマ:...`
- `/成績`
- `/復習`
- `/help`

役立つ場面：テスト勉強、資格学習、授業内容の復習。

工夫：問題を生成するだけでなく、Discord上の回答結果を集計し、その結果を次の出題へ反映する。

実装難易度：中〜高

## 候補の比較

| 候補 | 役立ち度 | Discordらしさ | AI以外の工夫 | 期限内の作りやすさ |
| --- | --- | --- | --- | --- |
| Discord議論ファシリテーター | 高 | 非常に高い | スレッド・履歴取得・ボタン投票 | 高 |
| タスク分解 | 高 | 高 | 担当・進捗管理 | 中 |
| 議事録 | 高 | 高 | 履歴取得・項目別表示 | 高 |
| FAQ | 高 | 高 | 類似検索・知識の再利用 | 低〜中 |
| クイズ・弱点分析 | 高 | 高 | 回答集計・弱点判定 | 中 |

## 企画検討時の推奨（現在は議事録Botを採用）

第一候補の「Discord議論ファシリテーターBot」を推奨する。

理由：

1. チャンネル、スレッド、メッセージ履歴、ユーザー、ボタンを一つの機能として利用できる
2. AIは意見の分類・要約を担当し、Botは議論場所の作成、履歴収集、投票、結果発表を担当するため、AIだけに依存しない
3. PostgreSQLなしでも課題条件を満たせる
4. スレッド作成とボタン投票は `discord.js` の機能で実装でき、短期間でも独自性のある最小構成を完成させやすい

## 第一候補の最小完成版（MVP）

1. `/議題` で現在のチャンネルに議論用スレッドを作る
2. 複数メンバーがスレッドへ普段どおり意見を投稿する
3. `/整理` でスレッドのメッセージ履歴を取得し、Geminiへ送って分類結果を表示する
4. 整理結果に投票用ボタンを付け、Discordユーザーごとに1票を記録する
5. `/結果` で投票結果を同じスレッドへ表示する
6. `/help` ですべてのコマンドと使用例を表示する
7. Expressの `/` で稼働状況を返し、Renderへデプロイする

余裕があれば、投票終了時の参加者通知、同票時の決選投票、PostgreSQLへの議論結果保存を追加する。スレッドとボタン投票は作品の独自性に関わるため、追加機能ではなく最小完成版に含める。
